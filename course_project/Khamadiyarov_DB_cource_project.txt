+ 1) Составить общее текстовое описание БД и решаемых ею задач;
В качестве курсового проекта спроектирована БД для сервиса-маркетплейса в сфере B2B.
Сервис предназначен для объединения компаний в едином информационном пространстве. В БД будет храниться информация о компаниях и их пользователях, запросах, предложениях, товарах, сделках.
+ 2) минимальное количество таблиц - 10;
    mysql> SHOW TABLES;
    +--------------------------+
    | Tables_in_marketplace    |
    +--------------------------+
    | business_lines           | 
    | companies                |
    | companies_business_lines |
    | deal_statuses            |
    | deals                    |
    | delivery_methods         |
    | file_types               |
    | files                    |
    | offers                   |
    | offers_files             |
    | products                 |
    | products_categories      |
    | products_files           |
    | requests                 |
    | requests_files           |                                                                                                                                 
    | users                    |                                                                                                                                 
    | users_logs               |                                                                                                                                 
    +--------------------------+
    
    companies - список компаний;
    business_lines - сферы деятельности компаний. Каждая компания может работать в нескольких сферах;
    companies_business_lines - таблица связей компаний и сфер деятельности;
    delivery_methods - способы доставки товаров или услуг компании;
    users - сотрудники компаний;
    files - файлы, загруженные сотрудниками компании при составлении запросов, предложений, при заведении товаров;
    file_types - таблица допустимых расширений файлов;
    products - товары компании;
    products_categories - категории товаров;
    products_files - таблица связей файлов и товаров;
    requests - запросы компании, запросы НЕ могут быть связаны с товарами;
    requests_files - таблица связи запросов и файлов;
    offers - предложения, предложения могут быть связаны с товарами;
    offers_files - таблица связи предложений и файлов;
    deals - таблица сделок (контрактов). Каждая сделка проводится на основании запроса (таблица requests) или предложения (таблица offers);
    users_logs - лог-таблица типа ARCHIVE для хранения действий добавления, обновления и удаления в некоторых таблицах (см. триггеры)
